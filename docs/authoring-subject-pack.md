# 교과 팩 작성 안내

새 교과 팩은 UI나 판정 컴포넌트를 수정하지 않고 `src/content/packs/<pack-id>` 폴더와 `registry.ts` 항목만 추가하는 것을 원칙으로 합니다.

## 1. 팩 메타데이터

- `id`: 비어 있지 않은 kebab-case 문자열
- `subjectLabel`: 학생 화면에 보일 교과 데스크 이름
- `shortDescription`: 사건에서 연습할 핵심을 한 문장으로 설명
- `displayOrder`: 데스크 정렬 순서
- `iconToken`, `accentToken`: 허용된 로컬 토큰만 사용
- `learningGoals`: 이 팩으로 직접 연습할 수 있는 목표만 기록

## 2. 사건과 주장 조각

사건은 반드시 `synthetic: true`이고 실제 기관·인물·언론사·URL을 포함하지 않습니다. 주장 조각은 대상, 시점, 수치, 범위, 사건, 관계, 의견을 구분합니다. 의견은 `checkable: false`로 두고 진위 판정에서 제외합니다.

처음 출처 ID는 정확히 4개, 후속 출처 ID는 1개를 지정합니다. 첫 판정과 최종 판정은 `confirmed`, `partly-confirmed`, `contradicted`, `insufficient` 중 하나입니다.

## 3. 출처 카드와 계보

모든 출처는 다음 정보를 학생에게 숨기지 않습니다.

- 작성 주체와 자료 유형
- 발행일 또는 `null`과 `날짜 표시 없음`
- 관찰·적용·기록 기간
- 수집·관찰·작성 방법
- 적용 가능한 대상·장소·기간·표본 범위
- 학생용 짧은 본문과 접근 가능한 요약

원자료를 옮긴 재게시물은 원자료와 같은 `originId`를 쓰고 `derivedFromId`를 지정합니다. 같은 `originId`는 독립 근거 한 개로만 계산합니다.

## 4. 출처×주장 관계

모든 확인 가능한 주장 조각마다 다음 관계 중 하나를 구조화된 값으로 작성합니다.

- `supports`: 해당 조각을 직접 지지
- `contradicts`: 같은 대상·시점의 결과가 반대
- `limits`: 일부 관련되지만 표본·기간·조건 때문에 범위를 좁혀야 함
- `irrelevant`: 현재 검증 질문에 직접 도움이 되지 않음

표시 문장을 파싱해 판정하지 않으므로 `assessments`를 빠뜨리면 팩 검증이 실패합니다.

## 5. 이유와 제목

`reasonOptions`에는 올바른 이유와 대표 오개념을 함께 두되, 체크포인트마다 정확히 선택해야 할 이유를 `correctAt`으로 표시합니다. 오개념 이유를 덧붙인 선택은 통과하지 않습니다.

제목은 자유 장문 자동 채점 대신 교과 검수를 거친 선택지를 두 개 이상 제공하는 것을 권장합니다. `무조건`, `전부`, `완벽`, `충격`, `100%` 같은 과장 표현을 정확한 제목에 쓰지 않습니다.

## 6. 안전·접근성 검수

- 정치·선거, 의료, 재난·범죄, 전쟁·혐오, 투자·가격 예측 소재를 사용하지 않습니다.
- 실제 학생·교사·학교에 관한 소문이나 개인정보를 쓰지 않습니다.
- 수치 자료는 같은 값을 담은 표와 텍스트 요약을 제공합니다.
- 색상만으로 관계나 선택 상태를 구분하지 않습니다.
- 성취기준을 연결할 때 앱이 실제로 지원하는 활동 범위와 한계를 함께 적습니다.

## 7. 검증

```bash
npm test -- tests/content tests/domain
npm run typecheck
npm run check:lines
```

팩 수를 고정한 UI 분기문을 만들지 말고 레지스트리 정렬 결과를 그대로 렌더링합니다.
