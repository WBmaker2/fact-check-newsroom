# 팩트체크 편집국 디자인 시스템

## 목적

팩트체크 편집국은 초등학생이 가상 자료를 차분히 읽고 판단하는 편집 활동이다. 시각 요소는 정답 경쟁보다 읽기, 비교, 실수 복구를 돕는 데 사용한다.

## 디자인 원칙

1. 편집국과 종이 문서의 인상을 유지한다.
2. 쉬운 말을 먼저 보여 주고 학습 용어는 보조로 표시한다.
3. 한 화면에서 가장 중요한 행동은 하나만 강하게 강조한다.
4. 상태는 색만으로 구분하지 않고 아이콘과 문장을 함께 사용한다.
5. 모바일에서는 모든 단계를 나열하지 않고 현재 단계와 전체 수를 보여 준다.
6. 점수, 순위, 과한 애니메이션보다 근거 읽기와 다시 보기를 우선한다.

## 색상

| 토큰 | 값 | 용도 |
|---|---|---|
| `--bg` | `#f7f4ed` | 종이색 전체 배경 |
| `--surface` | `#fffefa` | 자료 카드와 결과 카드 |
| `--fg` | `#1d2428` | 본문과 주요 테두리 |
| `--muted` | `#667078` | 보조 설명 |
| `--accent` | `#244f8f` | 현재 단계와 주요 행동 |
| `--success` | `#24684f` | 완료와 일치 |
| `--warning` | `#8a5c12` | 다시 확인할 내용 |
| `--danger` | `#a63b32` | 가상 자료 표시와 초기화 확인 |

색은 의미를 보조한다. 완료, 경고, 선택 상태에는 반드시 글자 또는 아이콘을 함께 둔다.

## 글꼴과 위계

- 큰 제목: `--font-display` 세리프 계열
- 본문과 버튼: `--font-body` 산세리프 계열
- 날짜, 단계, 짧은 분류표: `--font-mono`
- 한 문장은 가능한 한 한 행동만 안내한다.
- 판정은 `맞아요` 같은 쉬운 이름을 크게, `자료로 확인됨` 같은 학습 용어를 작게 표시한다.

## 간격과 화면 폭

- 본문 최대 폭: `1200px`
- 결과 카드 최대 폭: `880px`
- 기본 공간 단위: 4px 기반 `--space-*`
- 모바일 본문 좌우 여백: 12px 이상
- 터치 영역: 높이 44px 이상

## 주요 컴포넌트

### 단계 진행 표시

- 데스크톱: 다섯 단계 전체를 동일한 간격으로 표시한다.
- 모바일: `현재 단계/5`, 현재 이름, 진행 막대를 한 화면에 표시한다.
- 단계가 바뀌면 화면을 위로 이동하고 `<main>`에 초점을 보낸다.

### 행동 영역

- 기본 순서: `이전 단계 → 현재 상태 → 다음 단계`
- 모바일·짧은 화면에서는 화면 아래에 고정해 다음 행동을 찾기 쉽게 한다.
- 비활성 버튼 옆에는 남은 행동 수를 표시한다.

### 자료 카드

- 자료 번호, 자료 종류, 가상 자료 표시, 출처 이름, 핵심 문장을 순서대로 둔다.
- 네 가지 출처 단서는 버튼 한 번으로 펼치되 질문과 답을 모두 보이게 한다.
- 옮겨 적은 자료는 같은 원자료임을 별도 안내한다.

### 근거 비교

- 먼저 도움 되는 자료를 고른다.
- 선택한 자료만 확인할 말과 비교한다.
- 틀린 행만 배경과 문장으로 다시 확인하도록 안내한다.

### 판정

- 네 판정은 2×2 또는 모바일 1열로 표시한다.
- 판정 하나와 이유 하나만 고르게 한다.
- 맞힌 뒤 다음 화면에서 왜 맞는지와 다른 판정과의 차이를 짧게 설명한다.

## 접근성

- `aria-pressed`, `aria-current`, `aria-live`로 선택과 진행 상태를 전달한다.
- 모달을 닫으면 호출 버튼으로 초점을 돌린다.
- 단계 이동 시 새 화면의 메인 영역에 초점을 보낸다.
- 200% 확대와 320px 화면에서 내용과 행동 버튼이 겹치지 않아야 한다.
- 움직임 감소 설정에서는 전환 시간을 최소화한다.

## 인쇄

- 교사용 안내와 결과 카드는 서버 전송 없이 현재 브라우저에서만 인쇄한다.
- 인쇄 시 앱 헤더, 진행 표시, 조작 버튼은 숨긴다.
- 결과 카드에는 `교육용 가상 자료`와 판정 한계를 남긴다.

## 피해야 할 표현

- 실제 뉴스의 진위를 자동 판정하는 듯한 문장
- 사람이나 기관의 신뢰 점수
- 점수·순위·경쟁 중심 보상
- 여러 색의 카드와 배지를 한 화면에 과도하게 사용
- 저장하지 않으면서 저장된 것처럼 보이는 완료 표현
