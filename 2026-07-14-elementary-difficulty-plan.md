# 팩트체크 편집국 초등 난이도 개선 기획

## 목표

- 현재 5~6학년 중심의 활동을 초등 4~6학년이 교사 도움 없이도 진행할 수 있는 수준으로 낮춘다.
- 팩트체크의 핵심 학습 목표인 `주장 확인 → 출처 살피기 → 근거 비교 → 판정 → 정확한 제목` 흐름은 유지한다.
- 한 사건의 예상 활동 시간을 18~22분에서 약 10~15분으로 줄인다.

## 확인된 어려움

1. 한 사건에서 처음부터 자료 4개를 읽어야 해 읽기 부담이 크다.
2. 2단계의 `작성 주체·방법·범위`와 3단계의 `지지·반박·제한` 같은 용어가 초등학생에게 추상적이다.
3. 3단계에서 자료마다 여러 주장 조각을 다시 분류해야 해 선택 횟수가 많다.
4. 4단계에서 긴 주장, 네 개의 전문 판정명, 복수 정답 이유를 한꺼번에 판단해야 한다.
5. 오답 피드백이 일반적이어서 학생이 어느 자료를 다시 봐야 하는지 알기 어렵다.
6. 과학·통계 사건은 비율, 표본, 남중 고도처럼 여러 개념을 동시에 이해해야 한다.

## 추천 변경안

### 1. 전체 언어를 어린이 말로 바꾸기

- 전문 용어는 없애지 않고 쉬운 말을 먼저 보여 준다.
  - `자료로 확인됨` → `맞아요` + 보조 표기 `자료로 확인됨`
  - `일부만 확인됨` → `일부만 맞아요`
  - `제시된 자료와 맞지 않음` → `아니에요`
  - `근거가 부족해 판단 보류` → `아직 몰라요`
- `작성 주체·방법·범위`는 `누가 만들었나요?·어떻게 알아봤나요?·누구/어디 이야기인가요?`처럼 질문형으로 바꾼다.
- 한 안내 문장은 한 가지 행동만 말하고, 긴 문장은 두 문장으로 나눈다.

### 2. 읽고 누르는 양 줄이기

- 각 사건의 첫 자료를 4개에서 3개로 줄인다. 핵심 원자료, 헷갈리기 쉬운 자료, 관련이 적은 자료의 역할은 유지한다.
- 3단계 관계 선택지는 쉬운 표현으로 바꾸고, 현재 선택 횟수를 명확히 보여 준다.
- 판정 근거는 최대 2개만 고르게 해 자료를 많이 고르는 것이 정답이라는 오해를 줄인다.

### 3. 4단계를 한눈에 풀 수 있게 만들기

- 해결 순서를 `① 가장 믿을 만한 자료 읽기 → ② 쉬운 판정 고르기 → ③ 이유 하나 고르기`로 줄인다.
- 긴 주장 아래에 사건별 `핵심 비교` 한 줄을 표시한다.
  - 예: `도서관 원문: 첫째 토요일에 어린이 열람실을 엽니다.`
- 이유 선택은 복수 체크가 아닌 하나를 고르는 방식으로 바꾼다.
- 각 판정의 쉬운 이름을 크게, 학습 용어는 작게 표시한다.
- 틀렸을 때 정답을 바로 공개하지 않되, 다시 볼 자료나 숫자를 구체적으로 알려 준다.

### 4. 네 사건 내용 자체를 단순화하기

- 국어: 날짜와 장소가 원문 안내와 같은지만 비교하도록 문장을 짧게 만든다.
- 수학: `12명 중 10명`과 `5학년 전체`의 차이에만 집중하고 비율 계산은 요구하지 않는다.
- 과학: 남중 고도 개념을 빼고, 계절별 낮 길이 숫자 두 개를 직접 비교하게 한다.
- 사회: `처음 만듦`과 `고침`의 차이에만 집중한다.
- 주장, 출처 요약, 판정 이유, 최종 제목이 같은 어휘를 사용하도록 맞춘다.

### 5. 시작 화면과 진행 안내 조정

- 대상 표기를 `초등 4~6학년`으로 바꾸고 예상 시간을 `약 10~15분`으로 조정한다.
- 각 단계 제목에 학생이 해야 할 행동을 짧게 표시한다.
- 기존 업데이트 내역에 이번 난이도 개선 날짜와 핵심 변경을 추가한다.

## 수용 기준

- 초등학생에게 생소한 판정 용어는 모두 쉬운 말과 함께 제시된다.
- 처음 확인할 자료는 사건당 3개이며, 판정 근거는 1~2개만 선택할 수 있다.
- 4단계에서 학생은 판정 하나와 이유 하나만 선택하면 된다.
- 4단계에 사건별 핵심 비교 문장이 표시되고, 오답 시 구체적인 재확인 힌트가 나온다.
- 과학 사건은 남중 고도 지식 없이 표에 나온 낮 길이만으로 풀 수 있다.
- 네 사건 모두 첫 판정, 새 자료 확인, 최종 판정, 제목 선택까지 완료할 수 있다.
- 키보드 조작, 버튼 상태, 상태 안내 등 기존 접근성이 유지된다.
- 모든 변경 파일은 500줄 미만을 유지한다.
- 업데이트 내역에 `2026. 7. 14.` 초등 난이도 개선 항목이 추가된다.

## 지켜야 할 조건과 위험

- 네 가지 판정 개념과 첫 판정/최종 판정 비교는 학습 목표이므로 유지한다.
- 정답을 화면에 바로 노출하지 않고, 스스로 다시 찾을 수 있는 힌트까지만 제공한다.
- 읽기 부담을 낮추면서도 단순한 찍기 활동이 되지 않도록 핵심 원자료 비교는 남긴다.
- 저장 데이터나 외부 API는 없으므로 마이그레이션은 필요하지 않다.

## 검증 기록

- `npm test`: 40개 단위·컴포넌트 테스트 통과
- `npm run check`: 린트, 타입 검사, 40개 테스트, 프로덕션 빌드, 500줄 제한 통과
- `npm run test:e2e`: 23개 E2E 통과
- 국어·수학·과학·사회 네 사건을 시작부터 결과 카드까지 자동으로 완료하는 E2E를 추가했다.
- 320px, 375px, 768px 가로 스크롤 없음, 44px 터치 영역, 모달 초점 복귀, 외부 요청·브라우저 저장소 미사용을 확인했다.
- 첫 Vitest 시도에 사용한 `--runInBand`는 지원되지 않는 옵션이라 제거한 뒤 정상 통과했다.
- 첫 E2E에서 결과 카드의 `읽기 전용` 안내를 찾지 못한 1건을 발견해 문구를 복원한 뒤 전체 통과했다.
- Safari 계열 WebKit 실행을 별도로 시도했으나 로컬 Playwright WebKit 실행 파일이 설치되어 있지 않아 시작하지 못했다. 기존 Chromium 전체 검증은 통과했다.
- 첫 Pages 배포 후 라이브 HTML에서 검색 설명의 기존 `초등 5~6학년용` 문구를 발견해 `초등 4~6학년용`으로 수정하고 재배포했다.
