# 팩트체크 편집국

초등 5~6학년 학생이 교육용 가상 사건을 통해 주장, 출처, 근거, 판정의 관계를 연습하는 정적 React 웹 앱입니다. 실제 뉴스를 자동 판정하지 않으며, 화면에 제시된 합성 자료만 다룹니다.

## 수업 흐름

사건 하나당 약 18~22분을 예상합니다.

1. 네 교과 데스크 중 사건 하나를 선택합니다.
2. 주장 문장에서 자료로 확인할 조각을 찾습니다.
3. 출처의 작성 주체, 날짜, 방법, 범위를 확인합니다.
4. 출처와 주장 조각의 관계를 지지·반박·범위 제한·관련 없음으로 분류합니다.
5. 첫 판정 뒤 후속 제보를 검토하고 최종 판정을 내립니다.
6. 근거가 허용하는 범위의 정확한 제목과 읽기 전용 결과 카드를 완성합니다.

## 실행

```bash
npm install
npm run dev
```

## 품질 확인

```bash
npm run lint
npm run typecheck
npm test
npm run test:e2e
npm run build
npm run check:lines
```

단위·컴포넌트 테스트는 도메인 규칙, 콘텐츠 팩, 상태 흐름, 접근 가능한 UI를 검사합니다. Playwright 17개 시나리오는 대표 전체 흐름, 반응형, 키보드 모달, 외부 요청·저장소 부재를 확인합니다.

## 개인정보와 판정 한계

- 이름, 사진, 음성, URL, 파일을 입력받지 않습니다.
- `localStorage`, `sessionStorage`, 쿠키, 서버 API를 사용하지 않습니다.
- 새로고침하면 진행 상황이 초기화됩니다.
- 판정은 실제 세상의 최종 진실이 아니라 사건 파일에 제시된 가상 자료 기준입니다.
- 근거 부족과 직접 반박을 구분하며 사람이나 기관에 신뢰 점수를 붙이지 않습니다.

## 콘텐츠와 코드 구조

- 교과 팩은 `src/content/packs` 아래에서 사건과 출처를 분리합니다.
- 도메인 판정과 검증은 `src/domain`의 순수 함수가 담당합니다.
- React 상태는 `src/app/newsroom-reducer.ts`의 메모리 리듀서만 사용합니다.
- TypeScript, TSX, CSS, MJS 코드 파일은 모두 500줄 미만이어야 합니다.

새 팩 작성 방법은 [docs/authoring-subject-pack.md](docs/authoring-subject-pack.md)를 참고하세요.

## 알려진 제한

- 로그인, 교사 대시보드, 클라우드 저장, 실시간 협업은 없습니다.
- 실제 뉴스 검색, URL 입력, AI 진위 판정은 없습니다.
- 자유 장문 제목 대신 교과 검수를 거친 제목 선택지를 사용합니다.
- 배포와 외부 서비스 등록은 이번 MVP 범위에 포함하지 않습니다.
