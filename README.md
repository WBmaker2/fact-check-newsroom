# 팩트체크 편집국

초등 4~6학년 학생이 교육용 가상 사건을 통해 주장, 자료, 근거, 판정의 관계를 연습하는 정적 React 웹 앱입니다. 실제 뉴스를 자동 판정하지 않으며, 화면에 제시된 합성 자료만 다룹니다.

## 공개 앱

[팩트체크 편집국 실행하기](https://wbmaker2.github.io/fact-check-newsroom/)

## 수업 흐름

사건 하나당 약 10~15분을 예상합니다.

1. 네 교과 데스크 중 사건 하나를 선택합니다.
2. 주장 문장에서 자료로 확인할 말 두 개를 찾습니다.
3. 자료 세 개에서 버튼 한 번으로 누가, 언제, 어떻게, 어디에 대해 만들었는지 확인합니다.
4. 도움 되는 자료를 먼저 고르고, 그 자료와 확인할 말이 같은지 쉬운 표현으로 비교합니다.
5. `맞아요`, `일부만 맞아요`, `아니에요`, `아직 몰라요` 중 하나와 이유 하나를 고릅니다.
6. 새 자료로 다시 판단한 뒤, 자료와 맞는 제목과 결과 카드를 완성합니다.

이전 단계로 돌아가도 선택이 유지됩니다. 시작 화면의 `교사용 수업 안내`는 단계별 발문과 학습 목표를 한 장으로 인쇄할 수 있고, 완료한 결과 카드도 서버 전송 없이 현재 브라우저에서 인쇄할 수 있습니다.

## 실행

```bash
npm install
npm run dev
```

## 품질 확인

```bash
npm run lint
npm run typecheck
npm test
npm run test:e2e
npm run build
npm run check:lines
```

단위·컴포넌트 테스트는 도메인 규칙, 콘텐츠 팩, 상태 흐름, 접근 가능한 UI를 검사합니다. Playwright 시나리오는 대표 전체 흐름, 단계 전환 위치, 이전 단계 복구, 모바일 진행 표시, 키보드 모달, 외부 요청·저장소 부재를 확인합니다.

## 개인정보와 판정 한계

- 이름, 사진, 음성, URL, 파일을 입력받지 않습니다.
- `localStorage`, `sessionStorage`, 쿠키, 서버 API를 사용하지 않습니다.
- 새로고침하면 진행 상황이 초기화됩니다.
- 판정은 실제 세상의 최종 진실이 아니라 사건 파일에 제시된 가상 자료 기준입니다.
- 근거 부족과 직접 반박을 구분하며 사람이나 기관에 신뢰 점수를 붙이지 않습니다.

## 콘텐츠와 코드 구조

- 교과 팩은 `src/content/packs` 아래에서 사건과 출처를 분리합니다.
- 도메인 판정과 검증은 `src/domain`의 순수 함수가 담당합니다.
- React 상태는 `src/app/newsroom-reducer.ts`의 메모리 리듀서만 사용합니다.
- TypeScript, TSX, CSS, MJS 코드 파일은 모두 500줄 미만이어야 합니다.

새 팩 작성 방법은 [docs/authoring-subject-pack.md](docs/authoring-subject-pack.md)를 참고하세요.

## 알려진 제한

- 로그인, 교사 대시보드, 클라우드 저장, 실시간 협업은 없습니다.
- 실제 뉴스 검색, URL 입력, AI 진위 판정은 없습니다.
- 자유 장문 제목 대신 교과 검수를 거친 제목 선택지를 사용합니다.
- GitHub Pages의 정적 앱으로 제공되며 서버 저장이나 사용자 계정 기능은 없습니다.
